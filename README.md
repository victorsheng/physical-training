# physical-training

个人训练计划与训记（Xunji）Open API 操作笔记。

## 文件

- `plan-A.md` — A/B 双日全身轮替计划。动作名已全部对齐训记标准动作表，可直接写回被识别。
- `skills.md` — 训记训练数据 Open API skill 定义（鉴权、接口、读写规则、限频）。

## 训记 Open API 速查

Base URL: `https://trains.xunjiapp.cn`

| 用途 | 接口 |
| --- | --- |
| 读取训练 | `POST /api_trains_for_llm_v2` |
| 写回训练 | `POST /api_upsert_trains_for_llm_v2` |
| 动作目录（只读，259 个标准动作） | `POST /api_movement_catalog_for_llm_v2` |

鉴权：请求头 `Authorization: Bearer <Key>`（Key 在 App 内申请，勿入库/勿外泄）。

### curl 注意事项（踩坑记录）

- **必须加 `--compressed`**：响应是 gzip 编码，不解压会得到乱码二进制。
- **限频**：同一训练日约 15 秒内最多读一次（旧文档写 90s，实测返回 `too frequent, retry after Ns`）。批量逐天读取时每次请求间 `sleep 6` 以上较稳。
- 写回限频约 45 秒。

## 写回经验（重要）

- **不存在创建"个人模板"的接口**。探测过 templates/template/routines 等端点均 404。模板只能：① 本地用 `plan-A.md` 维护，每次按它生成当天训练写回；或 ② 把计划当作 `done:false` 的训练记录写回充当模板。
- **单日最多 4 条训练**；每条写回最多 15 个动作、每动作最多 20 组。
- **写回先用 `dry_run:true` 预演**，验证动作名能否识别，再 `dry_run:false` 正式写。
- **只带 `reps`、不带 `weight` 的占位组疑似会被服务端丢弃**：实测一次 `done:false` + 仅 reps 的写回，整条训练只保留了带 `time` 的"平板支撑"，其余 reps-only 动作全部丢失。下次占位写回时给每组补一个占位 `weight`（如 `"weight":"0"` 或合理初始重量）再试。
- 动作只传中文 `name`，不传内部 `key`；不确定中文名时先查动作目录，只从表里选，不编造。
- 更新旧训练保留 `localid`/`start`/`end`；不带 `localid` 即新建。写回 payload 不包含的旧训练不会被删除。
